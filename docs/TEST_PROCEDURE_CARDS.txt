2. OVERVIEW
--------------------------------------------------------------------------------

DESCRIPTION:
4ga Boards is a real-time collaborative project management application (kanban
board). This test procedure covers two core features: Create Card and
Drag-and-Drop Card.

- What problem does it solve?
  Teams need a way to create work items and organize them visually across
  workflow stages (lists) in real-time.

- Who are the users?
  Team members with EDITOR role on a board who manage tasks collaboratively.

- What are the key workflows?
  * User opens a board, clicks "Add card" on a list, types a name, presses
    Enter to create a card
  * User drags a card from one list and drops it into another list
  * User reorders cards within a list by dragging to a new position
  * Multiple users view the same board and see changes in real-time

4. WORKFLOWS
--------------------------------------------------------------------------------

WORKFLOW: CREATE CARD

  1. User navigates to board, clicks "Add card" on a list
  2. Inline textarea appears; user types card name
  3. User presses Enter (or Ctrl+Enter for auto-open)
  4. Client generates local ID, calculates gap-based position
  5. Client dispatches optimistic Redux action (card appears instantly)
  6. Client sends POST /api/lists/:listId/cards via WebSocket
  7. Server validates EDITOR role, persists card to PostgreSQL
  8. Server broadcasts 'cardCreate' socket event to board channel
  9. Server creates Action record (audit trail)
 10. Other connected users see card in real-time
 11. On success: client merges server data. On failure: rollback.

WORKFLOW: DRAG-AND-DROP CARD

  1. User picks up card (mouse down + drag)
  2. react-beautiful-dnd renders placeholder and animates cards
  3. User drops card on target (same list, different list, or cancels)
  4. handleDragEnd fires, parses destination list and index
  5. If same position or no destination: no-op
  6. Saga calculates position (midpoint between neighbors)
  7. Client sends PATCH /api/cards/:id with {listId, position}
  8. Server validates EDITOR role, updates record, recalculates if collision
  9. Server broadcasts 'cardUpdate' to board channel
 10. Server creates CARD_MOVE or CARD_TRANSFER Action record
 11. All board viewers see card move in real-time

7. TEST DATA
--------------------------------------------------------------------------------

REQUIRED DATA:
- User accounts: EDITOR, VIEWER, COMMENTER (all on same board)
- Project with at least 1 board
- Board with 3 lists: "To Do", "In Progress", "Done"
- 3-4 cards in "To Do" list
- At least 1 label on a card

EDGE / NEGATIVE DATA:
- Empty string card name
- Special characters: <script>alert('xss')</script>, unicode, 500+ chars
- Null / missing name field in API
- Invalid listId (non-existent)
- Position: 0, negative, extremely large
- Exhausted position gaps
