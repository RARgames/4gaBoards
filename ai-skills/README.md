# QA Skill Drafts

These local skill drafts show how the automation approach can scale when new features are added.

| Skill | Use |
| --- | --- |
| `qa-test-planning` | Turn feature notes into scoped E2E/API/component test plans. |
| `qa-browser-inspection` | Inspect app screens and convert observed behavior into stable selectors and page-object actions. |
| `qa-api-exploration` | Explore backend routes and build safe API setup/cleanup helpers. |
| `qa-ci-triage` | Triage CI failures, artifacts, traces, and flaky-test signals. |
| `qa-performance-analysis` | Design lightweight performance smoke checks and thresholds. |
| `qa-security-testing` | Guide authz, input validation, and security smoke-test planning. |

They are not required to run the Playwright suite; they are deliverable artifacts demonstrating how AI-assisted QA work could remain repeatable as the product grows.
